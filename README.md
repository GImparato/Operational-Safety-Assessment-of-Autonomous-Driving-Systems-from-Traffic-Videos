# Operational Safety Assessment of Autonomous Driving Systems from Traffic Videos

## Overview

This repository contains the implementation developed for the research project **Operational Safety Assessment of Autonomous Driving Systems from Traffic Videos**.

The proposed framework estimates the operational safety of autonomous driving systems by combining traffic video analysis, Large Multimodal Models (LMMs), Operational Design Domain (ODD) characterization, and advanced statistical sampling algorithms.

The repository provides the complete experimental pipeline adopted in the paper, from traffic video processing to the estimation of collision probability using multiple sampling strategies under different operational profiles.

## LMM Pipeline

This folder contains the complete pipeline responsible for transforming raw traffic videos into structured scenario datasets.

The pipeline performs:

* traffic video analysis;
* scenario extraction;
* trajectory processing;
* ODD feature extraction;
* risk-related feature computation;
* generation of the final dataset (`scenario_metrics.csv`).

The produced dataset represents the input for all sampling algorithms.

---

## Sampling Algorithms

This folder contains the complete implementation of all sampling techniques evaluated in the paper.

Implemented algorithms include:

* Simple Random Sampling (SRS)
* Sampling Using Probability Sampling (SUPS)
* Ranked Heterogeneous Clustering (RHC)
* Stratified Simple Random Sampling (SSRS)
* Two-Stage Unequal Probability Sampling (2UPS)
* DEEPEST

For each algorithm, the repository also provides the corresponding implementation supporting Operational Profiles.

The experimental framework supports:

### Operational Profiles

* **P0** – Uniform operational profile
* **P1** – Random operational profile
* **P2** – Weather-oriented operational profile
* **P3** – Urban complexity operational profile

### Auxiliary Variables

* TTC (Time-To-Collision)
* PC (Probability of Collision)
* ODD Risk Index

### Partitioning Strategies

* None
* ODD-based partitioning
* HDBSCAN clustering

The folder also includes the experimental drivers used to reproduce the results reported in the paper.

---

## Graphic Plotter

This folder contains a Streamlit-based visualization tool used to inspect and compare the experimental results.

The application supports the visualization of:

* collision probability estimates;
* estimation error;
* execution time;
* observed failures;
* comparisons among sampling algorithms;
* comparisons among operational profiles.

# Installation

Clone the repository:

```bash
git clone https://github.com/GImparato/Operational-Safety-Assessment-of-Autonomous-Driving-Systems-from-Traffic-Videos.git

cd Operational-Safety-Assessment-of-Autonomous-Driving-Systems-from-Traffic-Videos
```

Create a virtual environment:

```bash
python -m venv venv
```

Activate it.

Linux/macOS:

```bash
source venv/bin/activate
```

Windows:

```powershell
venv\Scripts\activate
```

Install the required dependencies:

```bash
pip install -r requirements.txt
```

# Dataset

The experiments are performed on the dataset generated by the LMM Pipeline.

Each row represents a traffic scenario extracted from the analyzed videos.

The dataset contains:

* collision outcome;
* Time-To-Collision (TTC);
* estimated Probability of Collision (PC);
* ODD descriptors;
* environmental conditions;
* traffic characteristics;
* additional risk-related features.

The final dataset is stored as:

```
scenario_metrics.csv
```

# Generated Results

The experimental drivers generate CSV files containing one row for each experimental repetition.

Each row stores:

* sampling method;
* operational profile;
* auxiliary variable;
* partitioning strategy;
* sampling budget;
* run identifier;
* estimated collision probability (`theta_hat`);
* true collision probability (`theta_true`);
* observed failures;
* execution time (when available).

These files are directly used to generate the plots presented in the paper.
